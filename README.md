Bem-vindo ao meu repositório de estudos do curso de Java, ministrado pelo Prof. Dr. Nelio Alves. Este projeto documenta minha jornada do zero ao nível avançado, focando em boas práticas, design de sistemas e as tecnologias mais recentes do mercado.

## 🎯 Objetivo do Repositório
Este espaço serve para armazenar todos os exercícios, materiais de apoio, diagramas UML e projetos práticos desenvolvidos durante o curso, garantindo uma base sólida em Programação Orientada a Objetos (POO).

---

## 🏗️ Estrutura e Tópicos

### ✅ Já Abordados / Em Progresso
- **Fundamentos e Sintaxe:** Lógica de programação, algoritmos e sintaxe Java 25.
- **POO Básica:** Classes, atributos, métodos, construtores, sobrecarga e encapsulamento.
- **Memória e Estruturas:** Arrays, coleções (List), e comportamento de memória.
- **Calendarios e Datas:** DateTime, LocalDateTime, Instant e Calendar(Java 08).

### ⏳ Próximos Passos (A Serem Abordados)
- **Tópicos Especiais:** Enumerações e Composição.
- **Tratamento de Exceções:** Blocos try-catch e exceções personalizadas.
- **Pilares da POO:** Herança, Polimorfismo e Interfaces.
- **Generics & Set/Map:** Trabalhando com tipos seguros e estruturas de dados avançadas.
- **Programação Funcional:** Expressões Lambda e Stream API.
- **I/O:** Manipulação de arquivos e persistência básica.
- **Acesso a Dados:** JDBC (SQL), JPA e Hibernate (ORM).
- **Ecossistema Spring:**
  - Spring Boot 4 (Desenvolvimento Web e REST APIs).
  - Spring Data JPA & Spring Data MongoDB (NoSQL).
- **Interface Gráfica:** JavaFX.

## 🛠️ Tecnologias e Ferramentas
O projeto é desenvolvido utilizando um ambiente híbrido para máxima produtividade:

* **Linguagem:** Java 25 (LTS)
* **Framework Principal:** Spring Boot 4
* **IDEs:** Visual Studio Code & IntelliJ IDEA
* **Gerenciador de Dependências:** Maven

## 📖 Metodologia
O curso segue uma abordagem de **enfoque prático**:
1.  **Teoria:** Explicação clara dos conceitos.
2.  **UML:** Visualização da arquitetura antes do código.
3.  **Implementação:** Codificação guiada e exercícios propostos.
4.  **Projetos Reais:** Construção de sistemas completos para portfólio.
